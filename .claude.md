## Development Rules

### 1. No Emojis in Code
- **Rule**: Never add emojis to code files (Python, TypeScript, JavaScript, SQL, etc.)
- **Why**: Keep code professional, clean, and free from unicode issues
- **Exception**: Emojis ARE allowed in user-facing text (activity feeds, notifications, UI labels)
- **Examples**:
  - ‚ùå Bad: `console.log("‚úÖ API call succeeded")`
  - ‚úÖ Good: `console.log("[SUCCESS] API call succeeded")`
  - ‚úÖ Good: Activity feed showing "üéâ Wojtek climbed to Emerald!" (user-facing content)

### 2. Explore, Plan, Code Methodology
Always follow this three-step workflow:

#### Step 1: EXPLORE
- Research and understand requirements thoroughly
- Read existing code and documentation
- Analyze API responses and data structures
- Ask clarifying questions if anything is unclear
- Use Task tool with Plan subagent for complex exploration

#### Step 2: PLAN
- Present a detailed implementation plan
- Use ExitPlanMode tool to get user approval
- Include: files to create/modify, approach, potential issues
- Wait for explicit approval before coding
- Never skip this step, even for "simple" changes

#### Step 3: CODE
- Only execute after plan approval
- Follow the approved plan exactly
- Update todos as work progresses
- Communicate any deviations from plan

**Important**: If user is in plan mode, NEVER make file changes without approval. Always present plan first.
